# 青天业务演示与验收路径

## 1. 文档用途

本文用于把当前青天评标系统的业务演示路径整理成可交接、可试用验收、可功能核对的最短链路，帮助非开发人员理解当前青天专项成果。

本文用于：
- 交接演示；
- 试用验收；
- 功能路径核对；
- 区分“可查看/可复制/可导出”的业务交付动作与“需单独授权”的运行态脚本；
- 帮助非开发人员理解当前青天专项成果。

本文只做演示与验收路径说明：
- 不改变运行逻辑；
- 不接核心评分主链；
- 不改变 data 写入结构；
- 不替代接口测试、CI 和 PR 记录；
- 文档检查、静态检查、mock 测试不需要启动服务，也不需要 `ollama serve`。

## 评分报告与证据链交付说明

评分报告 JSON、DOCX、latest report、evidence trace、scoring basis、analysis bundle、compare_report、优化清单复制/JSON 导出等交付物说明见：[青天评分报告与证据链交付说明](qingtian-report-evidence-delivery.md)。

该文档用于交付物核对、验收、归档和交接；不改变运行逻辑，不接核心评分主链；真实服务访问或真实 Ollama 调用前需单独授权。

## 2. 演示前置条件

- 后续开发与验证应使用新 clean worktree。
- 不处理旧工作区未跟踪文件。
- 不运行 `git clean`。
- 不写 `.env`。
- 不提交密钥。
- 演示真实页面或接口前，需要由用户单独授权启动服务。
- 真实 Ollama 调用前，才需要提醒用户在 2 号窗口运行 `ollama serve`。
- 文档阅读、静态检查、mock 测试不需要启动服务，不需要 Ollama。

## 3. 最短业务演示路径

| 步骤 | 演示动作 | 预期可见结果 | 验收点 | 边界说明 |
|------|----------|--------------|--------|----------|
| 1 | 创建项目 | 页面或接口返回项目名称、项目 ID，并可在项目列表中看到 | 项目名称正确，项目 ID 或项目列表可见 | 创建真实项目属于运行态业务动作，演示前需单独授权启动服务 |
| 2 | 上传材料 | 招标文件、清单、图纸、施组或施工组织设计文本等材料出现在上传状态、材料列表或解析结果中 | 上传状态明确，材料列表或解析结果可核对 | 上传真实材料会写 data，不在只读阶段执行 |
| 3 | 评分施组 | 评分入口或 score/rescore 路径生成评分报告、分项结果或最新报告 | 总分、分项结果、证据链或最新报告可见 | 不在本文档任务中改 `scorer.py` / `v2_scorer.py` |
| 4 | 生成对比报告 | compare_report 或对比报告显示问题项、优化建议、推荐改写或优化清单 | 优化清单字段可读，问题与建议可核对 | 真实生成对比报告依赖已启动服务和已有项目数据 |
| 5 | 复制优化清单 | 点击“复制优化清单”后，可获得结构清楚的复制文本 | 复制文本可读，包含问题、原文内容、直接替换文本 / 原位补充内容、怎么改、验收标准、执行检查表等字段 | 复制动作使用当前浏览器最近一次对比报告结果，不重新评分，不写 data |
| 6 | 导出优化清单 JSON | 点击“导出优化清单 JSON”后，可获得 JSON 文件或 JSON 内容 | JSON 内容可交付、可归档、可复核 | 不触发 rescore，不接 Ollama |
| 7 | Ollama 增强预览 | 点击“Ollama 增强预览”后，预览结果可复制、可导出 | 预览结果可读，复制和导出动作可核对 | 仅预览，不写入正式学习进化结果，不影响评分，不进入核心评分主链；真实调用前才需要 `ollama serve` |
| 8 | 报告交付 | 汇总评分报告、对比报告、优化清单、JSON、DOCX 或 analysis bundle 等交付物 | 交付物可下载、可复制、可归档或可交接 | 报告交付说明不改变评分、存储或脚本行为 |

## 4. 验收清单

| 验收项 | 通过标准 | 证据材料 | 不通过处理 |
|--------|----------|----------|------------|
| 项目可创建 | 项目名称、项目 ID 或项目列表可见 | 页面截图、接口返回或项目列表记录 | 先核对服务是否已授权启动，再检查项目创建入口 |
| 材料可上传 | 材料上传状态、材料列表或解析结果可见 | 上传记录、材料列表或页面结果 | 检查文件格式、项目选择和上传入口 |
| 施组可评分 | 评分报告、分项结果、证据链或最新报告可见 | 评分结果页面、JSON、DOCX 或 latest report | 只定位问题，不在本任务中修改 `scorer.py` / `v2_scorer.py` |
| 对比报告可生成 | compare_report 输出问题项、优化建议、推荐改写或优化清单 | 对比报告页面或接口结果 | 确认同一项目下已有多份可比较施组 |
| 优化清单可复制 | 复制文本字段结构清楚，可读、可转交 | 剪贴板文本或人工粘贴结果 | 先生成对比报告，再执行复制动作 |
| 优化清单 JSON 可导出 | JSON 文件或 JSON 内容可交付 | 导出的 JSON 文件或内容片段 | 先生成对比报告，再执行导出动作 |
| Ollama preview 边界清楚 | 明确仅预览，不写正式学习进化结果，不影响评分，不进入核心评分主链 | 页面说明、预览结果、边界文档 | 真实调用前确认 `ollama serve`，只读阶段不调用 |
| 报告交付路径清楚 | 评分报告、对比报告、优化清单、JSON、DOCX 或 analysis bundle 可归档或交接 | 交付文件、下载结果或复制文本 | 明确缺失的是报告生成、下载还是说明问题 |
| 不接核心评分主链 | 演示说明和验收记录均不把复制、导出、预览动作接入核心评分主链 | 文档说明和 PR 记录 | 若要接入评分主链，必须另行授权和设计 |
| 不改 `scorer.py` / `v2_scorer.py` / `storage.py` | 本专项不修改评分和存储核心文件 | git diff 文件列表 | 发现触碰即停止并拆分任务 |
| 不提交 `.env` / 密钥 | 不出现 `.env`、token、password、API key 明文提交 | git status、diff、PR 文件列表 | 立即停止提交，先移除敏感信息 |

## 5. 需单独授权的动作

以下动作不是默认只读动作，不能在只读阶段执行，需要用户明确授权：

- 启动服务；
- 停止服务；
- 重启服务；
- `doctor.sh`；
- `restart_server.sh`；
- `data_hygiene repair`；
- `e2e_api_flow.sh`；
- 真实 Ollama 调用；
- 写 data；
- 写 build/logs；
- reset / force push / `git clean`。

当前文档任务不执行上述动作，只说明其边界。

## 7. 推荐演示口径

- 当前系统已形成阶段标签和回退定位，可按 `v0.1.7` 到 `v0.1.11` 追溯阶段成果。
- 对比报告优化清单支持复制和 JSON 导出，便于转交编制人员和归档验收。
- Ollama preview 仅作为增强预览，不写正式学习进化结果，不影响评分。
- health / ready / self_check 的只读与运行态边界已区分，避免把运行态诊断误当静态检查。
- diagnostic scripts 的副作用边界已说明，doctor、restart、data_hygiene repair、e2e_api_flow 均需单独授权。
- 当前演示路径不接核心评分主链。
- 当前演示路径不改 `scorer.py` / `v2_scorer.py` / `storage.py`。

## 8. 后续推进建议

- 演示前先确认是否需要启动服务。
- 涉及真实 Ollama 调用前先运行 `ollama serve`。
- 新业务功能继续采用“小任务、单分支、单 PR、CI 全绿后合并、阶段标签”的节奏。
