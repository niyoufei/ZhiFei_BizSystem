# 青天评标系统 - 零代码操作手册（单步命令）

适用对象：不写代码、只需要稳定使用系统的同事。
目标：所有关键动作都用一条命令完成。

## 1) 启动与诊断

在项目根目录打开终端，优先用这 4 条：

```bash
make doctor    # 自动诊断（必要时自动重启）
make restart   # 强制后台重启服务
make status    # 查看服务状态（端口 + health）
make stop      # 停止服务
make daemon-start # macOS 常驻启动（建议长期运行时用）
make daemon-status # 查看常驻状态
make daemon-stop # 停止常驻服务
STRICT=1 make doctor   # 严格诊断：缺少关键V2接口直接判失败
make acceptance # 一键严格验收（最推荐）
make acceptance-fast # 快速严格验收（跳过测试）
```

补充说明：
- 如果项目在 `Desktop` 路径，macOS 可能阻止 launchd 读取该目录。
- 系统会默认直接回退为普通后台重启模式（避免等待 launchd 失败超时），并在 `make daemon-status` 里显示 `mode=fallback` 与原因。
- `make daemon-status` 在 fallback 模式且服务掉线时会自动重启一次（auto-heal）。
- 如果你只想看状态、不希望自动重启，可执行：`AUTO_HEAL=0 make daemon-status`
- 如需强制尝试 launchd，可执行：`FORCE_LAUNCHD=1 make daemon-start`
- 若你需要严格常驻 keepalive，建议将项目移动到例如 `~/Code/` 后再执行 `make daemon-start`。

页面地址：

```text
http://127.0.0.1:8000/
```

### 如果看到 `incompatible architecture` 报错

这是 Python 包架构和当前终端架构不一致。直接在项目目录执行：

```bash
rm -rf .venv
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
make restart
make status
```

若仍失败，先开原生终端再重试（不要用 Rosetta 终端）。

## 2) 一键生成可发给 ChatGPT 的分析材料

### 单项目分析包

```bash
make analysis-bundle PROJECT_ID=<项目ID>
```

默认输出路径：

```text
build/analysis_bundle_<项目ID>.md
```

### 全项目批量分析包

```bash
make analysis-bundle-all
```

默认输出目录：

```text
build/analysis_bundles/
```

## 3) 一键端到端回归（自检整套流程）

```bash
make e2e-flow
STRICT=1 make e2e-flow
make acceptance
make acceptance-fast
```

说明：
- 会自动创建测试项目、上传样例文件、评分、录入一条真实评标、学习进化、导出分析包。
- 若本地服务未启动，会自动尝试重启（仅 localhost 场景）。
- 默认模式会兼容旧接口（避免因历史版本直接中断）；`STRICT=1` 用于重构验收。

回归结果路径：

```text
build/e2e_flow/summary.json
build/acceptance_summary.json
```

## 4) 页面里的三个高频按钮（无需命令）

在首页「2) 选择项目」区域：
- `系统自检`：检查后端关键能力是否正常。
- `评分体系Markdown`：生成可直接复制给 ChatGPT 的评分体系文本。
- `下载分析包(.md)`：直接下载当前项目分析包。

## 5) 常见问题

### Q1. 访问 127.0.0.1:8000 失败

按顺序执行：

```bash
make doctor
make status
make restart
```

### Q2. 需要 API Key 的操作失败

给命令加环境变量（示例）：

```bash
API_KEY=<你的密钥> make e2e-flow
```

### Q3. 不知道项目ID

在页面点击 `刷新项目列表`，下拉框中可见项目名与ID前缀；也可调用：

```bash
curl -s http://127.0.0.1:8000/api/v1/projects
```

### Q4. 如何确认“当前运行的是新版本服务”

```bash
make doctor
make spec-coverage
```

说明：
- `make doctor` 会显示运行服务是否缺少关键 V2 端点。
- `make spec-coverage` 会生成本地代码覆盖报告：`build/v2_spec_coverage.md`。
