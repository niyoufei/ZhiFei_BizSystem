# 青天评标系统 - 产品说明与常见问题

## 核心价值：自我进化与两方向应用

本系统的核心价值在于**通过对实际得分的分析获得经验，驱动系统自我进化**，进化后的能力用于两个方向：

- **A、评分系统进化**：使本系统预评分更贴近**合肥市公共资源交易中心青天大模型**的评分规则。学习进化后会产出「维度权重建议」（scoring_evolution），并在后续项目施组评分时自动应用，从而让预评分更接近真实评标结果，对编制施组的预评分极具参考价值。
- **B、编制系统指令**：将学习到的高分逻辑与编制指导沉淀为**编制系统指令**（必备章节、必备图表/图片、必备要素、禁止表述等），可导出为文本或系统约束，用于强制要求按此指令输出施组内容与图表，指导如何编制获得高分的施组。

使用方式：录入真实评标（施组全文 + 5 评委分 + 最终得分）→ 点击「学习进化」→ 查看「高分逻辑」「编制指导」「评分进化（维度权重建议）」→ 使用「编制系统指令」查看/导出编制约束。后续上传施组评分时，将优先使用进化得到的维度权重，使预评分更贴近青天。**建议至少录入 3 条以上真实评标再做学习进化**，高/低分组对比更充分，维度权重建议更有效。

---

## 一、数据持久化与资料库

**创建项目、上传资料和施组后，数据会一直保存在本机**（`data/` 目录下的 JSON 与文件）。

**存在的意义：**

- **对比与洞察**：多份施组可做排名、维度对比、常见扣分统计。
- **学习画像**：根据历史提交生成维度权重，后续评分可做自适应微调。
- **自我学习与进化**：投喂包 + 真实评标录入后，系统学习高分逻辑并生成编制指导。
- **自适应建议**：基于历史扣分给出词库/规则优化建议，并可应用补丁。

若在系统完成学习和分析后，您希望清理单条记录以节省空间或合规，可使用**资料库**中的**删除**按钮，删除单条「项目资料」或「项目施组集」记录。
资料库入口：首页「资料库（当前项目）」→ 选择项目 → 刷新资料库 → 在表格中对单条执行删除。

## 二、上传格式说明

- **项目资料（分区上传）**：
  - 招标文件和答疑：`.txt/.md/.pdf/.doc/.docx/.json/.xls/.xlsx`
  - 工程量清单：`.xls/.xlsx/.csv/.pdf/.doc/.docx`
  - 图纸和标准资料：`.dxf/.dwg/.pdf/.doc/.docx/.xls/.xlsx/.png/.jpg/.jpeg/.webp/.bmp/.tif/.tiff`
  - 现场照片：`.png/.jpg/.jpeg/.webp/.bmp/.tif/.tiff`
- **施组文件**：支持 `.txt/.doc/.docx/.pdf/.json/.xls/.xlsx/.dxf/.dwg`。上传后进入**待评分**，点击「评分施组」才会出分。

---

## 三、系统是否具备「识别文件」和「对比文件」能力？

**具备。**

- **识别/分析**：对施组文本进行 16 维度评分、三步闭环逻辑锁、扣分项与证据片段提取；评分时会联合检索项目资料（招答/清单/图纸/现场照片）并注入运行时约束。
- **对比**：对同一项目下多份施组提交做「对比排名」「对比报告（叙述）」「洞察」（弱项维度、常见扣分、建议），以及「生成学习画像」等。

## 四、进化 LLM 配置与可追溯

- **配置**：通过环境变量 `EVOLUTION_LLM_BACKEND` 选择进化后端：`rules`（仅规则）、`openai`、`gemini`。历史 `spark` 仍可作为兼容别名输入，但会映射到 `openai`。配置对应 API Key（如 `OPENAI_API_KEY`、`GEMINI_API_KEY`）后，学习进化会由该模型增强高分逻辑与编制指导。
- **状态查询**：`GET /api/v1/config/llm_status` 返回当前进化后端及各 API 是否已配置（不暴露密钥），便于确认生效能力。
- **可追溯**：进化报告中的 `enhanced_by` 字段标识本次报告实际由哪个 LLM 后端增强（当前为 `openai` / `gemini`），便于审计与对比。评分报告 `meta.applied_multipliers` 记录本次评分应用的维度权重，可与进化产出的 `scoring_evolution` 对应。

---

## 五、接入不同 AI API 对分析能力和自我进化的影响

**会有影响。** 不同大模型在理解力、稳定性、对评标场景的适配上存在差异，会直接影响：

- 若启用 **LLM 评分/微调**（如当前的 OpenAI GPT-5.4）：分析深度、评语风格、与规则引擎的融合效果。
- **自我进化**（真实评标录入 + 学习进化）：若未来进化逻辑中引入 LLM（例如归纳高分逻辑、生成编制指导），则所选模型也会影响进化质量。

**当前与后续计划：**

- 当前系统已支持 **OpenAI GPT-5.4** 作为默认 LLM 评标/微调，并支持 **Gemini** 作为进化增强后端。
- 历史 `spark` 入口目前只是 OpenAI 的兼容别名，不再代表真实使用讯飞星火 API。

## 六、关于 Cursor「Auto 模式」与模型选择

**对「已部署运行的青天评标系统」没有影响。**
系统跑在您本机或服务器上的 Python + FastAPI，其能力由当前代码与配置（规则、词典、可选 OpenAI / Gemini API 等）决定，与您在 Cursor 里用哪种模式、哪个模型对话无关。

**对「搭建和修改系统」有影响：**
在 Cursor 中，不同模型会影响我（AI 助手）理解需求、写代码、排错和重构的质量与速度。若您**不在乎价格、更看重搭建与智能化改造效果**，通常选用**能力更强的模型**（例如 Cursor 提供的更高级别模型）会更好，便于完成复杂需求、多步修改和架构级调整。具体以 Cursor 当前提供的模型列表为准，选其标注为「更强」或「推荐用于复杂任务」的即可。

## 七、系统如何保留和升级自己获得的经验？

**会保留，并可升级。**

- **保留**：学习进化后，系统将高分逻辑、编制指导、评分进化（维度权重建议）、编制系统指令等写入本机 `data/evolution_reports.json`（按项目）；真实评标原始数据保存在 `data/ground_truth_scores.json`。这些数据会持久存在，供后续评分与编制指令使用。
- **升级**：再次在「自我学习与进化」中点击「学习进化」时，系统会基于当前项目已录入的真实评标重新计算，并覆盖该项目的进化报告。若配置了 `EVOLUTION_LLM_BACKEND=openai/gemini`，会用大模型在规则结果上做增强，使高分逻辑与编制指导更贴近自然语言与场景理解。若仍写 `spark`，系统会按兼容别名映射到 `openai`。
- **应用**：评分时优先使用进化得到的维度权重（使预评分更贴近青天）；编制系统指令可导出，作为编制施组时的约束。建议流程：选择项目 → 上传项目资料 → 录入真实评标 → 学习进化 → 查看/导出编制系统指令；之后上传施组即可享受进化后的预评分与指导。
