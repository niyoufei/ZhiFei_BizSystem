# 智飞施组评标评分系统 - 试用指南

本文帮助您快速启动系统并试用，同时列出试用后可能还需要修改的地方，便于您按需调整。

**本系统支持在 macOS 与 Windows 上安装并启动，同一套代码与数据格式，按您当前系统选择下方对应步骤即可。**

---

## 一、环境与安装

- **Python**：3.10 或以上（推荐 3.11/3.12），Windows 与 Mac 均需先安装 Python 并加入 PATH。
- **架构（仅 Mac）**：若为 Mac M 系列（arm64），出现 `mach-o file, but is an incompatible architecture (have 'x86_64', need 'arm64')` 时，需用本机 Python 新建虚拟环境并重装依赖：
  - **Mac**：`python3 -m venv .venv` → `source .venv/bin/activate` → `pip install -r requirements.txt` → `make run` 或 `python3 -m app.main`
  - **Windows**：`python -m venv .venv` → `.venv\Scripts\activate` → `pip install -r requirements.txt` → 双击 `scripts\start.bat` 或 `python -m app.main`

**首次安装依赖：**

| 系统 | 在项目目录执行 |
|------|----------------|
| **Windows** | `python -m pip install -r requirements.txt` |
| **macOS / Linux** | `pip3 install -r requirements.txt` 或 `make install` |

---

## 二、启动系统

按系统任选一种方式即可；启动后约 2.5 秒浏览器会自动打开系统页面（也可手动访问 http://localhost:8000/）。

| 系统 | 推荐方式 | 其他方式 |
|------|----------|----------|
| **Windows** | 双击 **`scripts\start.bat`** | 在项目目录 cmd 中执行 `python -m app.main` |
| **macOS / Linux** | `make run` | `bash scripts/start.sh` 或 `python3 -m app.main` |

启动成功时终端会提示：

```
浏览器将自动打开: http://127.0.0.1:8000/
按 Ctrl+C 停止
```

若 8000 端口被占用：
- **Windows**：用记事本打开 `scripts\start.bat`，在 `python -m app.main` 前加一行 `set PORT=8080`，保存后重启；浏览器访问 http://localhost:8080/
- **Mac**：在终端执行 `PORT=8080 make run` 或 `PORT=8080 python3 -m app.main`。

## 三、试用步骤（建议按顺序）

在浏览器打开 **http://localhost:8000/** 后，按页面从上到下操作即可。

1. **创建项目**
   输入项目名称（如「测试施组评审」）→ 点击「创建」。记下返回的 `project_id`（若需要）。

2. **选择项目**
   点击「刷新项目列表」→ 在下拉框选择刚创建的项目；或直接在「或直接输入 project_id」中粘贴上一步的 id。

3. **上传施组（仅上传）**
   在「施组文件」中选择本仓库自带的示例（如项目根目录的 `sample_shigong.txt`）或您自己的施组文件，点击「上传施组」。
   上传后列表显示“待评分”，不会自动出分。

4. **评分施组（手动触发）**
   点击「评分施组」，系统才会执行评分并刷新总分、规则分、证据链等结果；数据保存在本机 `data/` 目录。

5. **多份施组对比**
   对同一项目再上传 1～2 份施组（如 `sample_shigong_action_missing.txt`、`sample_shigong_empty_promises.txt`），然后：
   - **对比排名**：查看各次提交的分数对比
   - **对比报告（叙述）**：查看文字版对比分析
     - 生成对比报告后，页面会显示结构化优化清单，包含「问题」「原文内容」「直接替换文本 / 原位补充内容」「怎么改」「验收标准」「执行检查表」等字段。
     - 点击「复制优化清单」会复制当前页面内最近一次对比报告的优化清单文本。
     - 点击「导出优化清单 JSON」会导出当前页面内最近一次对比报告的 JSON 数据。
     - 如果尚未生成对比报告，点击复制或导出会提示：`请先生成对比报告。`
     - 复制 / 导出只读取当前浏览器前端内存中的最近一次 compare_report 结果，不会重新评分，不会触发 rescore，不会写入 data/，不会影响正式分数。
   - **洞察**：项目维度洞察
   - **生成学习画像**：学习画像报告

6. **自适应优化（可选）**
   有多次提交后可使用：
   - **自适应建议** → **生成补丁** → **验证效果**
   - **应用补丁** 需配置 API Key，为可选功能

7. **CLI 与批量（可选）**
   在项目目录执行（Windows 将 `python3` 改为 `python`，无 `make` 时见下方命令）：
   - 单文件评分：`python3 -m app.cli score -i sample_shigong.txt -o build/output.json --docx-out build/output.docx`
   - 批量（Mac）：`make batch`；Windows 或不用 make 时：`python -m app.cli batch -i sample_shigong.txt -o build/batch_output --docx`
   - 缓存预热：`python3 -m app.cli warmup -i filelist.txt`（filelist.txt 每行一条文本）

## 四、试用后可能还需要修改的地方

试用过程中若发现不符合预期，可重点从下面几类入手修改。

### 1. 界面与文案

- **位置**：`app/main.py` 中首页 HTML（约 1353 行起，`@app.get("/")` 下方的长字符串）。
- **可改**：标题「青天评标系统」、说明文案、按钮文字、步骤提示（如「首次使用：① 创建项目…」）等。
- **建议**：若有多处重复文案，可考虑抽到 `app/resources/locales/zh.yaml` 做 i18n，与现有中英文一致。

### 2. 端口与主机

- **端口**：默认 8000，在 `scripts/start.sh`、`scripts/start.bat` 或启动命令中通过环境变量 `PORT` 修改（见 [Windows安装与使用.md](Windows安装与使用.md)）。
- **绑定地址**：若需对外访问，在 `app/main.py` 的 `uvicorn.run` 或启动脚本中设置 `host="0.0.0.0"`（注意安全与防火墙）。

### 3. 评分规则与维度

- **规则与词典**：`app/resources/rubric.yaml`、`app/resources/lexicon.yaml`。
- **维度与逻辑**：`app/engine/dimensions.py`、`app/engine/scorer.py`、`app/engine/logic_lock.py`。
- 修改后重启服务即可生效；若启用了配置热加载，部分配置可通过「重载配置」类接口更新。

### 4. 报告与导出

- **DOCX 报告格式**：`app/engine/docx_exporter.py`、`app/engine/report_formatter.py`。
- **多语言**：`app/resources/locales/zh.yaml`、`en.yaml`，以及报告中的 i18n key。

### 5. 数据与存储

- **数据目录**：默认 `app/data/`（由 `app/storage.py` 的 `DATA_DIR` 等决定），项目、提交、材料、学习画像等 JSON 均在此。
- 备份或迁移时直接复制 `data/` 目录即可；若需改路径，可调整 `app/storage.py` 中的路径常量。

### 6. 错误提示与健壮性

- 试用时若遇到「上传失败」「对比无数据」等，可看浏览器控制台（F12）和终端日志。
- 常见情况：
  - 未选项目或 project_id 错误 → 先刷新并选择项目，或正确输入 project_id。
  - 上传文件格式 → 施组上传支持 .txt（以及接口若支持 .docx/.pdf 则需与前端/接口一致）。
- 若希望错误提示更友好，可在 `app/main.py` 中对应接口的异常处理里增加或调整返回文案。

### 7. 认证与限流

- **API Key**：`app/auth.py`；部分写操作可要求 `X-API-Key`，便于正式环境使用。
- **限流**：`app/rate_limit.py`；当前基础设施就绪，可根据需要开启或调整策略。

### 8. 监控与运维

- **健康检查**：`GET /health`、`GET /ready`。
- **指标**：`GET /metrics`（Prometheus）；Grafana/Alertmanager 配置在 `grafana/`、`prometheus/` 下，按需启用。
- **命令边界**：健康稳定运行相关命令请先查看 `docs/health-stability-runtime-boundaries.md`，区分只读检查、pytest/mock、服务启动、ops_agents、Ollama 真实调用。该说明不改变评分逻辑，不影响正式分数，不接核心评分主链。

## 五、快速自检命令

在项目根目录执行，用于确认环境与核心功能是否正常：

```bash
# 代码风格检查（Windows 将 python3 改为 python）
python3 -m ruff check app/ tests/

# 单元测试（Mac：make test；Windows：python -m pytest tests/ -v）
make test

# 单文件评分 + DOCX（不依赖 Web）
python3 -m app.cli score -i sample_shigong.txt -o build/output.json --docx-out build/output.docx

# 严格验收（doctor + e2e + mece审计 + 数据卫生 + spec覆盖）
make acceptance-fast

# 全项目 MECE 审计汇总
make mece-audit

# 数据卫生巡检（孤儿记录检查）
make data-hygiene

# 4智能体协同巡检（单轮）
make ops-agents-run

# 4智能体后台守护（持续巡检 + 自动修复）
make ops-agents-start
make ops-agents-status
make ops-agents-stop
```

若上述命令均通过，且能正常启动 `make run` 并访问 http://localhost:8000/，则说明系统已就绪，可按第三节步骤试用；试用中遇到的具体问题可对照第四节定位修改。

## 六、若无法启动（常见原因）

- **报错含 `incompatible architecture (have 'x86_64', need 'arm64')`**
  当前 Python 为 arm64（如 M 系列 Mac），但已安装的包是 x86_64。解决：用本机 Python 创建新 venv，在 venv 内执行 `pip install -r requirements.txt` 后再 `make run`。

- **端口 8000 已被占用**
  修改端口（见 [Windows安装与使用.md](Windows安装与使用.md)）或关闭占用 8000 的程序。

- **依赖缺失**
  重新执行 `pip3 install -r requirements.txt` 或 `make install`。
